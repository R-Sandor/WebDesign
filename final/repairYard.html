Project:

Repair Yard

Description:

In this project for CS361(Algorithms and data strcutures) we were asked to create a military repair yard simulation. New vehicles would arrive every 24 hours and the program would decide which parts were good from the vehicle if any. The program would then decide whether the vechile was worth the estimated hours of work or if it was better scrap it for parts. In which case the vechicle would be thrown away and it's parts would be added to inventory. There were three repair bays that worked on all 24 hours. When a vechicle was repaired it would then be tested with 90% pass rate. If the vechile failed it would go back in the priority queue of vechiles which was based on the number of hours. The vehicles that required less time would be fixed first. If a vechile sat in the queue without a part for longer than 3 days it was scrapped for parts. A yard master ran the yard and made sure on every hour that the bays were making progress(fixing and testing), and odering new tanks to be assesed and repaired tanks to go leave for operations. On the end of each day the yard master would give report.

Objectives:

  • Using Priority Queues
  • Practicing good ADT design
  • Working in teams!
